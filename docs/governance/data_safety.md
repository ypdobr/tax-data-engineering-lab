# Data Safety Rules

This lab is built for public portfolio use. Every artifact must pass this check before it is committed or published.

## Hard Rules

- Use synthetic data only.
- Keep company, entity, supplier, customer and product names fictional.
- Use invented transaction IDs, tax codes, country mixes and amounts.
- Avoid copying Deloitte or client workflow names, templates, screenshots, files, report layouts or XML structures.
- Avoid using employer-owned Alteryx workflows, macros, Python snippets, prompt chains or documentation.
- Keep examples generic enough that they describe a professional capability, not a confidential delivery asset.

## Red Flags

Do not commit files that include:

- real legal entity names
- real VAT IDs, supplier IDs or account numbers
- real ERP extract names from client projects
- client-specific column names
- screenshots from work systems
- internal Deloitte wording or methodology
- real project documentation
- client-specific regulatory positions

## Safe Public Framing

Use wording like:

- ERP-style transaction data
- intercompany transaction review
- regulatory reporting package
- validation rule
- exception queue
- review note
- documentation draft

Avoid wording that implies copied delivery assets or sole ownership of employer IP.
